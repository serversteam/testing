Creating a CSV-file

Programs such as Excel have proven themselves for the management of tabular data. Excel can also import and process data in many formats. This is why this chapter describes how you can create a valid CSV file for the OpenEMM using Microsoft Excel.

In the Excel table, each recipient has an own line. Each column corresponds to a field in the recipient database. The first line of the CSV file contains the field labels (see figure below). You must enter the field names in upper case exactly as in OpenEMM. In the cells themselves you may use both upper and lower case.

Internally, OpenEMM manages all email addresses in small letters. When entering the email address you do not therefore have to take capitalization into account and all addresses can be entered directly using small letters.

Fig. 4.24: Every line in Excel matches a recipient, every column a field.

Important: Please note that the columns EMAIL and if necessary GENDER and MAILTYPE are mandatory. All other columns are optional and can be set up as desired. In the column GENDER enter the gender of the recipient with 0 (male), 1 (female) and 2 (unknown). In the column MAILTYPE enter numbers or text in the same way to indicate whether the recipient is to receive the text version (0, txt or text), the HTML version (1 or HTML), the Offline-HTML version (2) or the M-HTML version (4).

When all recipient data is entered into your spreadsheet software, save the data in CSV format. To do so, called the menu point File/Save in Excel. Set the desired target directory. In the Data Type selection list, set CSV (or *.csv depending on the Excel version). Save stores the CSV data on the hard disk. Excel may point out that not all worksheets can be saved in CSV format. Since this does not play a role, confirm with OK.

The CSV file that results is a simple text file which can be opened using any text program. The data for each recipient is stored in an own cell. The fields are marked by a separator character, for Excel this is a semicolon.
